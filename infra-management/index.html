Infrastructure Management

This section will explain how to manage Kubernetes/Docker infrastructure. This guide assumes that the necessary tools are already installed. For a quick intro to K8s/Docker and a deeper dive into how the infrastructure works, please see Architecture.

Infrastructure Makefile

Silimate Platform comes with a Makefile that contains all the typical commands necessary to manage the platform infrastructure. This Makefile is located in /silimate/platform-k8s on the CAD server. You can navigate to this directory and run make <command-name> to run a command.

Platform Bringup from Nothing

To bring up the platform infrastructure from nothing (from reboot for example), follow the steps below in order: - Start Minikube - Start NFS Server - Start Local Services - (Optional) Start K8s Dashboard

Additionally, it is quite useful to familiarize yourself with the Utility Commands and Miscellaneous Commands when debugging.

Starting Minikube

make start-minikube

If you are using minikube for Kubernetes (K8s), you can start it with this command. Under the hood, this runs:

sudo minikube start --memory=max --cpus=max --driver=docker --force

This effectively launches Minikube with no RAM/CPU limits using the Docker driver, ignoring any system incompatibility warnings (e.g., CentOS does not like Docker...)

To stop Minikube, use make stop-minikube or sudo minikube stop.

Starting NFS Server

make start-nfs-server

This launches the NFS server required to give filesystem access to Docker containers running in K8s.

To stop the NFS server, use make stop-nfs-server or sudo docker stop nfs-server.

Starting Local Services

make start

This launches all the services needed to run the platform.

To stop all the services, run make stop.

Starting K8s Dashboard

make start-k8s-dash

This launches the K8s dashboard.

To stop the K8s dashboard, run make stop-k8s-dash.

Utility Commands

Get Endpoint

Get the endpoint associated with the Platform K8s service.

make get-endpoint

Get Pods

Get the pod name that is produced by the K8s deployment.

make get-pods

Get Events

Very useful command for debugging. Gets the most recent Kubernetes events.

make get-events

Watch

Get events in a loop.

make watch

Debug

Run an alpine container inside of K8s to do any debugging of the infrastructure.

make debug
